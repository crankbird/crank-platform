# Philosophical Tagging Schema for Hybrid Integration

## Core Principle Detection

For each zettel, analyze content for manifestations of the Master Principle:  
**"Intelligence Is Situated in Space and Time"**

## Tagging Framework

### Primary Philosophical Markers

#### Space-Has-Meaning (SHM)
**Indicators:**
- Context-dependent behavior or outcomes
- Local-first technical architecture  
- Geographic/jurisdictional considerations
- Embodied cognition or situated intelligence
- Environmental context shaping possibilities
- Edge computing or distributed processing

**Examples:**
- "Different contexts produce different results"
- "Local processing reduces latency"
- "Regulatory environment shapes options"
- "Where you are changes what's possible"

#### Time-Unevenly-Distributed (TUD)  
**Indicators:**
- Temporal relativity in capability development
- Future emerging in pockets first
- Asynchronous systems or processes
- Historical context dependency
- Innovation diffusion patterns
- Temporal arbitrage opportunities

**Examples:**
- "Some teams living in the future"
- "Capabilities don't distribute evenly"
- "Early adopters vs mainstream adoption"
- "When you are changes what's real"

#### Identity-Is-Plural (IIP)
**Indicators:**
- Multiple identity systems or personas
- Context-dependent identity expression
- Role-based behavior variation
- Multiplicity-aware design
- Mascot or character systems
- Situational identity adaptation

**Examples:**
- "Different person in different contexts"
- "Persona-driven interface design" 
- "Role-based access control"
- "You are different depending on where you stand"

#### Agency-Is-Distributed (AID)
**Indicators:**
- Autonomous agents or systems
- Decentralized decision-making
- M2M economy or agent commerce
- Edge intelligence deployment
- Distributed coordination mechanisms
- Anti-centralization patterns

**Examples:**
- "Agents making autonomous decisions"
- "Edge nodes coordinating without center"
- "Distributed teams self-organizing"
- "Intelligence emerges across nodes"

#### Data-Has-Gravity (DHG)
**Indicators:**
- Computational locality principles
- Data sovereignty considerations
- Meaning-driven processing decisions
- Context-aware computation
- Local data preference patterns
- Gravity-based architecture decisions

**Examples:**
- "Process data where it lives"
- "Local context determines compute placement"
- "Data residency requirements"
- "Computation follows meaning"

## Tagging Methodology

### Analysis Process
1. **Read Complete Zettel** - Understand full context and meaning
2. **Identify Primary Markers** - Look for clear manifestations of core principles
3. **Note Secondary Themes** - Catalog supporting concepts and applications
4. **Rate Philosophical Coherence** - Scale 1-5 for worldview alignment
5. **Document Connections** - Note links to other zettels or concepts

### Tagging Format
```yaml
philosophical_analysis:
  primary_markers:
    - SHM: 4  # Strong space-has-meaning content
    - TUD: 2  # Weak temporal relativity
    - AID: 5  # Very strong distributed agency
  secondary_themes:
    - TECH: 3  # Moderate technical content
    - BIZ: 1   # Minimal business application
  coherence_score: 4
  connections:
    - "Links to distributed systems architecture"
    - "Supports M2M economy vision"
  synthesis_notes: >
    Strong example of distributed agency principle applied to technical
    architecture. Demonstrates how philosophical worldview translates into
    concrete implementation choices.
```

### Integration Examples

#### High Philosophical Coherence (Score 4-5)
Zettels that strongly express core principles across multiple domains.
